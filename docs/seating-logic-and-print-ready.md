# تطوير التسكين والمنطقيات والطباعة

## المكونات الجديدة
- شاشة `منطقيات التسكين`: المسار `/seating/logic`
- تحسينات `إدارة المقاعد`: فلترة الصورة + بحث منطقي + حل "اجعله يجلس بجانب..."
- تحسينات `الجاهزين للطباعة`: فلترة جاهز/غير جاهز + أولوية تغييرات المقاعد + مؤشر أصفر + سجل زمني

## واجهات API الجديدة

### 1) تقرير التعارضات المنطقية
- `GET /seating/logic/report?eventId=MINYA-2026-MAIN`
- يعيد قائمة الأزواج/المجموعات التي لديها تفضيل مجاورة لكنها غير متجاورة فعلياً.

### 2) الحل الذكي لإعادة التسكين
- `POST /seating/logic/solve`

#### Body
```json
{
  "event_id": "MINYA-2026-MAIN",
  "attendee_ids": ["id-1", "id-2"]
}
```

#### المنطق
- يحاول إيجاد كتلة مقاعد متجاورة فارغة للمجموعة المطلوبة.
- إذا لم تتوفر، يحاول إخلاء كتلة متجاورة عبر نقل الأشخاص الموجودين عليها إلى مقاعد منفردة متاحة.
- ثم يعيد تسكين المجموعة المستهدفة مع تحديث المقعد/الباركود.

## تتبع تغييرات المقاعد للطباعة

### حقول تمت إضافتها (منطقياً عبر metadata fallback)
- `seat_change_pending`
- `seat_change_last_at`
- `last_seat_code`
- `seat_change_history[]`

### سلوك العمل
- عند تغيير مقعد شخص من كود مقعد إلى كود مختلف، يتم:
  - رفع `seat_change_pending = true`
  - إضافة سجل زمني إلى `seat_change_history`
- عند `mark-printed` (تيكت أو شهادة):
  - يتم مسح المؤشر تلقائياً `seat_change_pending = false`

## أمثلة استخدام من الواجهة
- من صفحة `منطقيات التسكين`:
  - "حل" لكل تعارض
  - "حل الكل"
  - "اجعله يجلس بجانب..." عبر اختيار شخص ثم شخص آخر
- من صفحة `إدارة المقاعد`:
  - بحث باسم الشخص/ID/الهاتف
  - عرض الجيران الحاليين
  - تنفيذ حل منطقي مباشر

## ملاحظات أداء
- تم تصميم الحل للعمل على أعداد كبيرة عبر:
  - ترتيب وفرز مقاعد بشكل deterministic
  - حل بالمجموعات
  - تقليل عمليات القراءة المتكررة قدر الإمكان
